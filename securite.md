# Bonnes pratiques de sécurité

## 1. Utiliser un mot de passe fort
Un mot de passe fort contient au moins 12 caractères, des majuscules, des chiffres et des symboles.

## 2. Ne jamais partager son token GitHub
Le Personal Access Token est un secret. Il ne doit jamais être communiqué ni commité dans le dépôt.

## 3. Activer l'authentification à deux facteurs (2FA)
La 2FA protège le compte GitHub même si le mot de passe est compromis.
